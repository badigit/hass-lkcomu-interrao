# Инструкции по разработке интеграции lkcomu_interrao

Этот файл содержит контекст и инструкции для работы с кодовой базой кастомной интеграции `lkcomu_interrao` для Home Assistant.

## Обзор проекта

`lkcomu_interrao` — это интеграция для Home Assistant, предназначенная для получения данных из различных личных кабинетов энергосбытовых компаний, входящих в группу «Интер РАО» (например, Мосэнергосбыт, МосОблЕИРЦ, Саратовэнерго и др.).

### Основные технологии
- **Язык:** Python 3.12+ (согласно `pyproject.toml`).
- **Фреймворк:** Home Assistant Core.
- **Библиотека API:** `inter-rao-energosbyt-python`.
- **Валидация:** `voluptuous` (стандарт HA).

### Архитектура
- **`custom_components/lkcomu_interrao/`**: Основная директория интеграции.
  - `__init__.py`: Инициализация интеграции, настройка YAML и `config_entry`.
  - `_base.py`: Базовые классы и логика обновления данных (делегаты обновлений).
  - `sensor.py`, `binary_sensor.py`: Реализация сущностей Home Assistant.
  - `config_flow.py`: Логика настройки через интерфейс (UI).
  - `const.py`: Константы проекта (домен, ключи конфигурации, форматы имен).
  - `_util.py`: Вспомогательные функции (маскирование логинов, работа с иконками, импорт классов API).
  - `_schema.py`: Схемы валидации конфигурации.

## Разработка и тестирование

### Установка зависимостей
Для локальной разработки рекомендуется использовать `pip` и виртуальное окружение:
```bash
pip install -r requirements.txt
# Или напрямую зависимости из manifest.json:
pip install inter-rao-energosbyt-python==0.0.23 fake_useragent==0.1.11 python-dateutil==2.8.1
```

### Форматирование кода
Проект использует `black` для форматирования кода. Параметры указаны в `pyproject.toml`:
- Длина строки: 88 символов.
- Целевые версии Python: 3.11, 3.12.
- Команда для запуска: `black .`

### Тестирование
В проекте не обнаружено явных папок с тестами (например, `tests/`). Тестирование обычно проводится путем загрузки интеграции в экземпляр Home Assistant.

## Конвенции и стандарты

1.  **Логирование**: Используйте `_LOGGER` с префиксами, генерируемыми через `_make_log_prefix` из `_util.py`. Это помогает идентифицировать, к какому аккаунту относится сообщение.
2.  **Интернационализация**: Проект поддерживает русский и английский языки. Проверяйте флаг `IS_IN_RUSSIA` (в `_util.py`) для вывода соответствующих сообщений в логи. Переводы для интерфейса находятся в `translations/`.
3.  **Сущности**: Все новые датчики должны наследоваться от `LkcomuInterRAOEntity` из `_base.py`.
4.  **Безопасность**: Никогда не логируйте пароли или полные номера лицевых счетов. Используйте `mask_username` для логинов в логах.
5.  **Конфликты слияния**: В репозитории могут присутствовать маркеры конфликтов (`<<<<<<< HEAD`). Перед внесением изменений убедитесь, что файлы очищены от них.
